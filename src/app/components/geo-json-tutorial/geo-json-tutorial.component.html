GeoJSON Tutorial This page introduces GeoJSON.

Some tab content

GeoJSON is a format used to represent geographic data. A GeoJSON data stucture is a basic JSON object. An object is made of name-value pairs. The GeoJSON object must have a 'type' value, which is a string that determines the type of the GeoJSON object. It could be either geometry, a feature or a collection of features. The GeoJSON also have a 'geometry' value that is a geometry object. Each geometry object has a 'type' and 'coordinates' value. 

                        {{ '{' }}
                        "type": "Feature",
                        "geometry": {{ '{' }}
                          "type": "Point",
                          "coordinates": [125.6, 10.1]
                          {{ '}' }},
                        "properties": {{ '{' }}
                          "name": "Dinagat Islands"
                         {{ '}' }}
                      {{ '}' }}

Some more tab content

A geometry value describes the type of the GeoJSON object. It could be "Point", "MultiPoint", "Line String", "MultiLineString", "Polygon", "MultiPolygon", "GeometryCollection". Every one of this types except for the GeometryCollection must have a "coordinates" value.

  • "Point" - the "coordinates" value must be a single position
  • "MultiPoint" - the "coordinates" value must be an array of positions
  • "Line String" - the "coordinates" value must be an array of two or more positions
  • "MultiLineString" - the "coordinates" value must be an array of LineString coordinate arrays
  • "Polygon" - the "coordinates" value must be an array of LinearRing coordinate arrays
  • "MultiPolygon" - the "coordinates" value must be an array of Polygon coordinate array
  • "GeometryCollection" - doesn't have a value for coordinate, instead it represents a collection of geometry objects

A feature object also contains a "geometry" value. Also it has a "properties" value.
