GeoJSON Tutorial This page introduces GeoJSON.

GeoJSON is a format used to represent geographic data. A GeoJSON data stucture is a basic JSON object. An object is made of name-value pairs. The GeoJSON object must have a 'type' value, which is a string that determines the type of the GeoJSON object. It could be either geometry, a feature or a collection of features. The GeoJSON also have a 'geometry' value that is a geometry object. Each geometry object has a 'type' and 'coordinates' value. The 'coordinates' value is a decimal number, which represents a dimension, typically the longitude and latitude. GeoJSON data structure is used as a programatic interface for applications. The syntax of GeoJSON is shown below. 

                        {{ '{' }}
                        "type": "Feature",
                        "geometry": {{ '{' }}
                          "type": "Point",
                          "coordinates": [longitude, latitude, depth]
                          {{ '}' }},
                        "properties": {{ '{' }}
                          "name": "string"
                         {{ '}' }}
                      {{ '}' }}

A geometry value describes the type/shape of the GeoJSON object. Every geometry, except for the GeometryCollection, has a 'type' and 'coordinates' value. It could be "Point", "MultiPoint", "Line String", "MultiLineString", "Polygon", "MultiPolygon", "GeometryCollection".

"Point" - the "coordinates" value must be a single position "MultiPoint" - the "coordinates" value must be an array of positions "Line String" - the "coordinates" value must be an array of two or more positions "MultiLineString" - the "coordinates" value must be an array of LineString coordinate arrays "Polygon" - the "coordinates" value must be an array of LinearRing coordinate arrays "MultiPolygon" - the "coordinates" value must be an array of Polygon coordinate array "GeometryCollection" - doesn't have a value for coordinate, instead it represents a collection of geometry objects

A feature object also contains a "geometry" value. I t has a "properties" value, that can be any kind of JSON object. The syntax is shown below.

                      {{ '{' }}
                      "type": "Feature",
                      "geometry": {{ '{' }}
                        "type": "Point",
                        "coordinates": [longitude, latitude, depth]
                        {{ '}' }},
                      "properties": {{ '{' }}
                        "name": "string"
                       {{ '}' }}
                    {{ '}' }}

A FeatureCollection object has a "type" of "FeatureCollection" and must contain an array of objects of type "feature".

                      Example:
            
                      {{ '{' }} "type": "FeatureCollection",
                              "features": [
                              {{ '{' }} "type": "Feature",
                                  "geometry": {{ '{' }}"type": "Point", "coordinates": [longitude,
                                  latitude,
                                  depth] {{ '}' }},
                                  "properties": {{ '{' }}"prop0": "value0" {{ '}' }}
                                  {{ '}' }},
                                  {{ '{' }} "type": "Feature",
                                  "geometry": {{ '{' }}
                                    "type": "LineString",
                                    "coordinates": [
                                      [0, 0], [0, 0], [0, 0], [0, 0]
                                      ]
                                      {{ '}' }},
                                  "properties": {{ '{' }}
                                    "prop0": "value0",
                                    "prop1": 0.0
                                    {{ '}' }}
                                    {{ '}' }},
                                
                                ]
                                {{ '}' }}
